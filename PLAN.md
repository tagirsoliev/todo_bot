# План реализации: Telegram-бот «Напоминалка» — версия 1

## Что делаем в v1

Telegram-бот, где каждый пользователь из белого списка может:
- добавлять задачи себе;
- отправлять задачи другому пользователю (выбор получателя кнопкой);
- смотреть свой список, отмечать «Сделал ✅», редактировать и удалять свои задачи.

Админ дополнительно может:
- нажать «Отправить всем» → каждый получатель получает свой индивидуальный список невыполненных задач;
- управлять белым списком (добавить / удалить пользователя по Telegram-ID).

## Чего в v1 НЕТ (добавим при соединении с тудушником)

- Автоматическая почасовая рассылка (её пока заменяет ручная кнопка «Отправить всем»).
- Тихие часы 22:00–08:00 (Ташкент).

Обе вещи закладываем в код заранее (функции написаны, но не вызываются планировщиком), чтобы при соединении просто «включить».

## Технологический стек (решено, менять не нужно)

| Слой            | Выбор                    | Почему                                                        |
|-----------------|--------------------------|--------------------------------------------------------------|
| Язык            | TypeScript (Node.js)     | Тудошник на Next.js — при соединении код ляжет почти без правок |
| Telegram        | grammY                   | Современная TS-библиотека, чистый API, ложится и в скрипт, и в webhook |
| База            | Neon (Postgres)          | Бесплатно, авто-просыпание <1 c, родная интеграция с Vercel   |
| ORM             | Drizzle ORM              | Лёгкий, TS-native, схему потом легко шарить с тудушником      |
| Запуск v1       | long polling             | Не нужен публичный URL; при соединении переключим на webhook  |

## Схема базы данных

### `users`
| Поле         | Тип        | Описание                                  |
|--------------|------------|-------------------------------------------|
| id           | serial PK  | Внутренний id                             |
| telegram_id  | bigint     | Telegram-ID пользователя (уникальный)     |
| name         | text       | Имя (из Telegram, для выбора получателя)  |
| is_admin     | boolean    | Админ ли (по умолчанию false)             |
| created_at   | timestamptz| Когда добавлен                            |

Только пользователи из этой таблицы могут пользоваться ботом (это и есть белый список).

### `tasks`
| Поле         | Тип        | Описание                                       |
|--------------|------------|------------------------------------------------|
| id           | serial PK  | Id задачи                                      |
| text         | text       | Текст задачи                                   |
| owner_id     | bigint     | Кому адресована (кто её видит и выполняет)      |
| author_id    | bigint     | Кто создал (для прав на редактирование/удаление) |
| is_done      | boolean    | Выполнена ли (по умолчанию false)              |
| created_at   | timestamptz| Когда создана                                  |

Логика ролей на одной задаче:
- **owner** — получатель, видит задачу в своём списке, жмёт «Сделал».
- **author** — создатель, может редактировать/удалять (даже если owner — другой человек).
- Если добавляешь задачу себе → owner_id == author_id.

Ключ архитектуры: **вся логика рассылки живёт в `services/reminders.ts`**. Кнопка «Отправить всем» просто вызывает эту функцию. Когда добавим cron — он вызовет ту же функцию. Никакого дублирования, соединение проходит гладко.

## Пользовательские сценарии (диалоги)

**Добавить задачу себе**
1. Пользователь: `/add` (или кнопка «➕ Добавить»).
2. Бот: «Кому задача?» → кнопки [Себе] [Другому].
3. «Себе» → «Напишите текст задачи».
4. Пользователь пишет текст → задача сохранена (owner = author = он сам).

**Отправить задачу другому**
1. То же меню → «Другому».
2. Бот показывает кнопки со списком пользователей белого списка.
3. Пользователь выбирает получателя → «Напишите текст задачи».
4. Текст → задача сохранена (owner = получатель, author = отправитель).

**Мой список**
1. `/list` (или кнопка «📋 Мои задачи»).
2. Бот выводит невыполненные задачи, под каждой: [✅ Сделал] [✏️ Изменить] [🗑 Удалить].
   («Изменить»/«Удалить» показываются, только если пользователь — author задачи.)

**Отправить всем (только админ)**
1. Админ: кнопка «📢 Отправить всем».
2. Бот проходит по всем пользователям, каждому шлёт его невыполненные задачи (или «Все задачи выполнены 👍»).
3. Админу — краткий отчёт: скольким отправлено.

**Управление белым списком (только админ)**
- `/adduser <telegram_id> <имя>` — добавить.
- `/removeuser <telegram_id>` — удалить (с подтверждением).
- `/users` — показать список.

8. **Управление белым списком** (админ-команды).

## Задел под соединение с тудушником (чтобы ты понимал будущее)

При соединении поменяется немного:
- **Запуск:** long polling → webhook (grammY webhookCallback как Next.js API-роут).
- **Планировщик:** внешний cron-job.org дёргает роут `/api/reminders` раз в час; внутри включается проверка тихих часов (код уже готов).
- **База:** та же самая Neon, данные не переезжают.

Бизнес-логика (services/*) и схема базы остаются как есть. Поэтому v1 пишем аккуратно с разделением «логика ↔ способ запуска».
